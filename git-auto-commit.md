请作为专业的全栈开发工程师，根据以下要求，完成一个在线挂机系统的开发手册编写和完整代码实现。

## 任务目标
开发一个 7×24 小时在线挂机系统，能够自动、持续地向指定 GitHub 仓库生成提交记录，模拟真实、活跃的开发历史。你需要同时输出完整的项目代码和一份详细的开发手册（Markdown 格式）。

## 背景与参考项目
请参考两个 9 年前的老旧项目思路，但必须使用现代、安全、易维护的方案进行重构：
- https://github.com/tywei90/git-auto-commit
- https://github.com/linux4cn/git-and-python

旧项目的核心思路：通过脚本定时修改仓库中的某个文件（如日志文件），然后自动 `commit` 并 `push`，从而制造提交记录。请继承这种思想，但需要做出以下现代化改进：
- 更智能的随机化策略（提交时间、消息、文件内容都不可预测）
- 完全基于配置文件驱动，无需硬编码
- 使用 GitHub Personal Access Token 进行安全认证，并安全存储（环境变量、`.env` 文件，不提交到仓库）
- 支持多种部署方式，以适应不同的挂机环境

## 系统功能要求
1. **持续运行能力**：系统可以 7×24 小时运行（或按计划触发），在规定的时间窗口内自动执行提交和推送。
2. **高度可配置**：至少支持以下配置项：
   - 目标仓库地址（支持 HTTPS 带 token 的方式）
   - 目标分支名称
   - 每日提交次数范围（如 0~8 次，随机取值）
   - 每次提交的最短、最长间隔（单位：分钟）
   - 提交信息模板或一个可自定义的消息列表，支持随机选取
   - 要修改的文件名或文件列表（如 `activity.log`、`README.md` 更新日期等）
3. **模拟真实行为**：
   - 提交时间随机化，避免整点或固定频率
   - 提交消息使用真实感的句子（可参考旧项目中“上班摸鱼”“自动更新”等风格，可自定义列表）
   - 文件修改内容应有变化（追加时间戳、随机数、随机选取的短语或表情符号等），避免完全相同的提交
4. **代码质量**：
   - 优先使用 Go 或 Python，代码结构清晰、模块化
   - 需要支持Windows、Linux和Macos等操作系统
   - 良好的注释和类型提示（如适用）
   - 错误处理和重试机制（网络异常、push 冲突等）
   - 日志输出，便于排查问题

## 开发手册要求
你需要生成一份完整的 `DEVELOPMENT_MANUAL.md`，包含以下章节：
- 系统概述与目标
- 技术选型说明（语言、库、依赖）
- 系统架构与工作流程（可用文字或 Mermaid 图描述）
- 目录结构说明
- 核心模块与代码逻辑详解
- 配置文件详细说明（每个参数的含义、默认值、示例）
- 部署指南，必须覆盖以下几种方式：
  - 本地开发与测试运行
  - 使用 Windows/Linux/macOS 的 `cron` 定时执行
  - 使用 GitHub Actions 定时自动提交（推荐作为挂机方案）
  - 使用 Docker 及 docker-compose 部署
- 安全注意事项（如何生成和配置 GitHub token，最小权限原则，如何保护 `.env` 文件）
- 常见问题与排错

## 交付物
